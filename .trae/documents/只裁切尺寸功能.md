# 只裁切尺寸功能（不换底）

## Context（背景）

用户希望在 Web UI 增加"只改尺寸、不换底"的功能：上传图片 → 选择尺寸 → 生成。保留原图背景，仅按证件照比例基于人脸位置裁剪并 resize 到目标尺寸。

**现状**：框架底层 `IDCreator.__call__` 已支持 `crop_only=True` 参数（[creator/__init__.py:58](file:///d:/HivisionIDPhotos/HivisionIDPhotos-master/hivision/creator/__init__.py#L58)），`inference.py` 和 `deploy_api.py` 都在用，但 Web UI（`processor.py`）从未传过该参数，所以这个能力对网页用户不可见。

**为何 Web UI 没暴露**：crop_only 模式下 `ctx.matting_image = ctx.processing_image`（3通道BGR原图），但下游 `IDphotos_cut`（[photo_adjuster.py:190](file:///d:/HivisionIDPhotos/HivisionIDPhotos-master/hivision/creator/photo_adjuster.py#L190)）创建4通道BGRA，`get_box`（[utils.py:69](file:///d:/HivisionIDPhotos/HivisionIDPhotos-master/hivision/creator/utils.py#L69)）甚至有4通道断言——3通道输入会直接 `ValueError` 崩溃。inference.py 因输入用 `IMREAD_UNCHANGED` 且常碰巧是4通道PNG而未暴露此 bug。

## 修改方案（4个文件）

### 1. 修复 crop_only 的 alpha 通道崩溃 — `hivision/creator/__init__.py`

[第132-133行](file:///d:/HivisionIDPhotos/HivisionIDPhotos-master/hivision/creator/__init__.py#L132-L133) crop_only 分支，给原图补一个全不透明（alpha=255）的 alpha 通道，使 matting_image 变成4通道，让 `IDphotos_cut` / `get_box` 正常工作：

```python
else:
    # crop_only: 保留原图背景, 补全不透明 alpha 通道供 adjust_photo 的 4 通道流程使用
    src = ctx.processing_image
    if src.shape[2] == 3:
        alpha = np.full((src.shape[0], src.shape[1]), 255, dtype=np.uint8)
        ctx.matting_image = cv2.merge((src, alpha))
    else:
        ctx.matting_image = src
```

全不透明 alpha 符合语义：crop_only 不抠图，人像和背景都是真实的、完全不透明的。

### 2. 多语言文案 — `demo/locales.py`

[size_mode](file:///d:/HivisionIDPhotos/HivisionIDPhotos-master/demo/locales.py#L90) 的 choices 列表（中/英/日/韩）末尾追加一项：
- 中文：`"只裁切尺寸"`
- 英文：`"Crop Only"`
- 日/韩对应翻译

新增项成为 `choices[4]`，不影响现有索引（`[0]`尺寸列表 / `[1]`只换底 / `[2]`自定义px / `[3]`自定义mm）。

### 3. 前端交互 — `demo/ui.py`

[change_size_mode 函数（第591-624行）](file:///d:/HivisionIDPhotos/HivisionIDPhotos-master/demo/ui.py#L591-L624) 加一个"只裁切"分支：
- 显示 `size_list_row`（尺寸列表，因为要选目标尺寸）—— 同"尺寸列表"模式
- 隐藏 `color_options`（背景颜色）和 `render_options`（渲染模式）—— 不换底不需要
- 保留 `plugin_options` 可交互

需要把 `color_options`、`render_options` 加入 `change_size_mode` 的返回 dict（所有4个现有分支都要补上 `gr.update(visible=...)`，保持其他模式行为不变）。

事件绑定处（change_size_mode 的 `.then` outputs 列表）补上这两个组件。

### 4. 后端处理 — `demo/processor.py`

**4a. 尺寸解析** — [_process_size_mode（第197-240行）](file:///d:/HivisionIDPhotos/HivisionIDPhotos-master/demo/processor.py#L197-L240)

"只裁切"模式需要尺寸（和"尺寸列表"/"自定义"一样解析），不能像"只换底"那样设 `(None, None)`。在现有判断里，把"只裁切"当作有尺寸的模式处理（复用 choices[0]/[2]/[3] 的尺寸解析逻辑）。

**4b. 传 crop_only** — [_generate_id_photo（第304-320行）](file:///d:/HivisionIDPhotos/HivisionIDPhotos-master/demo/processor.py#L304-L320)

```python
crop_only = (
    idphoto_json["size_mode"] == LOCALES["size_mode"][language]["choices"][4]
)
return creator(
    input_image,
    change_bg_only=change_bg_only,
    crop_only=crop_only,          # 新增
    size=idphoto_json["size"],
    ...
)
```

**4c. 跳过换底** — [_render_background（第408-452行）](file:///d:/HivisionIDPhotos/HivisionIDPhotos-master/demo/processor.py#L408-L452)

crop_only 模式不调 `add_background`。但 adjust_photo 输出的是 RGBA（人像区 alpha=255 原图背景，裁剪框超出区 alpha=0 透明），需把透明区合成白底。复用 [photo_adjuster.py 的 _composite_to_white（第153行）](file:///d:/HivisionIDPhotos/HivisionIDPhotos-master/hivision/creator/photo_adjuster.py#L153)：

```python
if crop_only:
    result_image_standard = _composite_to_white(result_image_standard)
    result_image_hd = _composite_to_white(result_image_hd)
    return result_image_standard, result_image_hd
# 否则走原 add_background 换底流程
```

**4d. 排版照** — [_generate_image_layout（第462-464行）](file:///d:/HivisionIDPhotos/HivisionIDPhotos-master/demo/processor.py#L462-L464)

当前只有"只换底"跳过排版。crop_only 有尺寸，正常生成排版照，无需改动。

## 不需要改动的部分

- `inference.py` / `deploy_api.py`：已有 crop_only 入口，本次只动 Web UI 路径
- `photo_adjuster.py`：`_composite_to_white` 已存在，直接复用
- 美颜/水印/人脸对齐：crop_only 模式下正常生效（用户可能需要美颜但不换底）

## 验证方式

1. 重启服务 `python app.py --port 7860`
2. 上传一张已有证件照（如蓝底1寸照）
3. 尺寸模式选"只裁切尺寸"，尺寸列表选"二寸"
4. 点击生成 → 验证输出是二寸尺寸（413×579）、保留蓝色背景、按人脸比例裁剪
5. 对比验证：同一张图用正常模式（抠图换底）确认两者行为不同
6. 边界测试：上传3通道JPEG（非PNG），确认不崩溃（验证 alpha 修复）
7. 确认"尺寸列表"/"只换底"/"自定义"等原有模式行为不变
